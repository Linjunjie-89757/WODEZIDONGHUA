# 工作空间架构设计说明

## 📋 概念区分

### 1. 工作空间（Workspace）
**定义**：实际的资源隔离单位，用于组织和管理测试资源

**位置**：`配置中心 → 空间配置`

**作用**：
- 资源隔离：每个空间有独立的测试用例、缺陷、执行记录
- 权限管理：每个空间有独立的成员和权限设置
- 数据隔离：不同空间的数据互不干扰

**示例**：
- 开户工作空间
- 个人工作空间
- 支付模块测试

### 2. 空间视图/过滤器
**定义**：用于切换查看不同范围数据的虚拟视图

**位置**：顶部导航栏的空间切换器

**作用**：
- 过滤数据范围
- "全部"视图：查看所有工作空间的汇总数据
- 具体空间：只查看该空间的数据

```
│   ├─ ───────────                                    │
│   ├─ 开户工作空间 ← 从配置中心读取                   │
│   ├─ 个人工作空间 ← 从配置中心读取                   │
│   └─ 支付模块测试 ← 从配置中心读取                   │

                过滤效果
                    ↓
┌─────────────────────────────────────────────────────┐
│              各功能页面数据显示                       │

```

```typescript

PUT    /api/workspaces/:id      // 更新工作空间
```

```typescript

```

## 🎯 设计原则

1. **单一数据源**：工作空间配置统一在配置中心管理
2. **虚拟视图**："全部"是视图层概念，不是数据层实体
3. **自动同步**：配置中心的变更自动反映到切换器
4. **清晰分离**：配置管理 vs 数据过滤 分离
