依赖方向固定为：

app
  -> pages
    -> widgets
      -> features
        -> entities
          -> shared
规则：

shared 不能依赖任何业务模块。
entities 不能依赖 features/widgets/pages。
features 不能依赖 widgets/pages。
widgets 可以组合 features/entities/shared。
pages 只负责页面组合，尽量不写复杂业务逻辑。
processes 只放跨多步骤的复杂流程，比如 AI 生成、接口执行流程。
新项目里最重要的纪律：

页面不要再写成 100KB 大文件。
不直接到处裸写 Arco 组件，优先用 shared/ui。
样式优先用 token 和 shared/ui，页面 scoped CSS 只做局部布局。
每个模块都要有 index.ts 对外导出，避免跨目录乱 import。
每迁移一个模块就做截图验收。
